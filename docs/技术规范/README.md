## ✍️ 如何编写技术规范

### 规范要求

1. **清晰的结构**：使用标准 Markdown 格式，层级清晰
2. **具体的例子**：代码示例要包含"正确"和"错误"做法
3. **详细的解释**：解释为什么要这样做，而不仅仅是做什么
4. **对比分析**：提供修改前后的对比
5. **参考资源**：链接相关的官方文档
6. **实用性**：包含最佳实践和常见问题

### 推荐的文档模板

```markdown
# [功能/问题] 技术规范

## 📌 文档信息
- 文档标题：
- 版本：
- 最后更新：
- 适用范围：
- 关键词：

## 🎯 问题概述
### 问题现象
### 根本原因分析

## ✅ 解决方案
### 方案概述
### 具体改动

## 📊 对比分析

## 🔧 实现细节

## 🎨 最佳实践

## 📚 参考资源

## 🚀 后续优化方向

## ❓ 常见问题

## 📝 修改日志
```
