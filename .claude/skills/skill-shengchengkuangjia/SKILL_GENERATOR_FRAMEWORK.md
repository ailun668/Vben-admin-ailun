# Claude Skill 生成框架

## 🎯 框架概述

一个**通用的、可复用的Skill生成系统**，能够快速创建专业级别的Claude Skill。

### 核心理念

```
用户输入 (角色/领域信息)
    ↓
交互式问卷 (系统化收集信息)
    ↓
框架模板 (结构化处理)
    ↓
自动生成 (SKILL.md + scripts + reference)
    ↓
质量检查 (验证完整性)
    ↓
完成的Skill (立即可用)
```

---

## 📋 生成流程

### 第一阶段：信息收集（交互式问卷）

系统会通过**5个核心维度**的问卷收集信息：

#### 1️⃣ **角色定义** (Role Definition)
```
问题：
- 这个角色的官方职称？
- 主要工作职责是什么？（5-7个）
- 核心技能要求？（5-7个）
- 日常工作的主要挑战？
```

#### 2️⃣ **知识体系** (Knowledge Domain)
```
问题：
- 这个角色需要掌握的关键概念？（10-15个）
- 最佳实践和常见模式？（5-10个）
- 行业标准和规范？（如有）
- 关键工具和方法论？
```

#### 3️⃣ **实战场景** (Use Cases)
```
问题：
- 这个角色最常见的工作场景？（5-7个）
- 每个场景的完整工作流程？
- 关键决策点和注意事项？
- 常见的失败案例？
```

#### 4️⃣ **工具和方法** (Tools & Methods)
```
问题：
- 常用的分析工具？
- 常用的方法论？
- 模板和框架？
- 自动化工具？
```

#### 5️⃣ **特殊需求** (Special Requirements)
```
问题：
- 行业特定的合规要求？
- 市场特定的考虑因素？
- 公司特定的流程？
- 其他特殊需求？
```

---

### 第二阶段：框架生成

基于收集的信息，系统自动生成完整的Skill结构：

```
Generated_Skill/
│
├── SKILL.md (1,200+ 行)
│   ├── 第1-3章: 基础知识和核心概念
│   ├── 第4-7章: 工作流程和决策框架
│   ├── 第8章: 完整案例研究
│   ├── 第9章: 工具和方法论
│   ├── 第10章: 常见问题和陷阱
│   ├── 第11章: 总结和最佳实践
│   └── 附录: 模板和检查清单
│
├── scripts/ (900+ 行)
│   ├── role_initializer.py     # 工作初始化脚本
│   └── quality_checker.py       # 质量检查脚本
│
├── reference/ (1,500+ 行)
│   ├── core_concepts.md         # 核心概念详解
│   ├── frameworks_models.md     # 框架和模型
│   ├── case_studies.md          # 案例研究集
│   ├── templates.md             # 工作模板
│   └── checklists.md            # 检查清单
│
└── TOOL_INTEGRATION.md          # MCP工具集成
```

## 📊 框架的关键特点

### 1. **标准化结构**
```
所有生成的Skill都遵循相同的结构：
- SKILL.md (1,200+ 行, 13章)
- scripts/ (2-3个工具脚本)
- reference/ (4-6个参考文档)
- 配置和说明文档
```

### 2. **质量保证**
```
自动检查清单：
- [ ] SKILL.md完整性 (章节数、代码示例、案例)
- [ ] 代码示例可运行性
- [ ] 参考文档准确性
- [ ] 脚本功能完整性
- [ ] 中文内容准确性
```

### 3. **可重复性**
```
同样的框架可以生成：
- 产品经理Skill ✅
- 工程经理Skill ✅
- 设计师Skill ✅
- 运营Skill ✅
- 任何其他角色Skill ✅
```

### 4. **可定制性**
```
不同行业、公司、阶段可以定制：
- 行业特定的案例
- 公司特定的流程
- 阶段特定的需求
- 文化特定的最佳实践
```
